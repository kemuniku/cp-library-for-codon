# 二部グラフマッチング

L個の左側頂点と、R個の右側頂点の間の最大マッチングを構築します。    
[最大流(Dinic法)](/algorithm/maxflow/)でも最大マッチングはできますが、こちらのほうが定数倍に優れます。

## 使い方

`BipartiteMatching(L: int, R: int) -> None`
- 左側頂点がL個・右側頂点がR個からなる、L + R頂点0辺の二部グラフを作成します。
- 制約: 0 ≤ L, R < $2^{30}$

`add_edge(Lt: int, Rt: int) -> None`
- 左側頂点Ltと、右側頂点Rtの間に無向辺を追加します。
- 制約: 0 ≤ Lt < L, 0 ≤ Rt < R

`build() -> None`
- これまでに追加した辺数をEとして、L + R頂点E辺の二部グラフにおける最大マッチングを構築します。
- **2026/01/01現在、この関数の返り値はNoneです。マッチング結果は`self.matched`に格納されます**。  
`matched[Lt]`は、左側頂点Ltがアンマッチであれば-1、マッチすれば対応する右側頂点Rt(0 ≤ Rt < R)を取ります。
- 計算量: V = min(L, R) として O(E√V + L + R)
